创建版本库
mkdir 【版本库名】
cd 【版本库名】
pwd 显示当前路径
git init  把这个目录变成Git可以管理的仓库

git add readme.txt
git commit -m "第一次使用git添加了一个文本文件"

git log [--pretty=oneline]  ---->打印日志【简易显示】以便确定要回退到哪个版本
git reset --hard HEAD^  ----->回退一个版本（回退两个就打两个^^）
git reset --hard [ecdea42]  ---->前进或回退到【版本号】

git reflog用来记录你的每一次命令，以便确定要回到未来的哪个版本。
eg：
ecdea42 (HEAD -> master) HEAD@{0}: reset: moving to ecdea42
c822db7 HEAD@{1}: reset: moving to HEAD^
ecdea42 (HEAD -> master) HEAD@{2}: commit: 第二次提交添加了一些内容
c822db7 HEAD@{3}: commit (initial): 第一次使用git添加了一个文本文件

如果是要重复修改 则：
第一次修改 -> git add -> 第二次修改 -> git add -> git commit

在还没有add之前，想要撤销修改则使用：
git checkout -- readme.txt

命令git checkout -- readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：
一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。
总之，就是让这个文件回到最近一次git commit或git add时的状态。

如果add了没commit想要撤销则：
git reset HEAD readme.txt
命令git reset HEAD <file>可以把暂存区的修改撤销掉（unstage），重新放回工作区
git reset命令既可以回退版本，也可以把暂存区的修改回退到工作区。当我们用HEAD时，表示最新的版本。

场景1：当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令git checkout -- file。

场景2：当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用命令git reset HEAD <file>，就回到了场景1，第二步按场景1操作。

删除文件：git rm 

要关联一个远程库：
git remote add origin git@github.com:【github用户名】/【仓库名】.git
添加后，远程库的名字就是origin，这是Git默认的叫法，也可以改成别的，但是origin这个名字一看就知道是远程库。

使用命令git push -u origin master 把本地master分支的最新修改推送至GitHub
此后，每次本地提交后，只要有必要，就可以使用命令git push origin master推送最新修改；
只要本地作了提交，就可以通过命令：git push origin master 推送至github

把github的仓库克隆到本地
git clone git@github.com:【github用户名】/【仓库名称】.git（使用默认协议git）
git clone https://github.com/【github用户名】/【仓库名称】.git（使用https协议）
